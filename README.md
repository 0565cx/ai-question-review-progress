> 北森 · AI 面试官 · 专业能力 —— 各岗位题目审核（审题）协作仓库
>
> 参照企业微信共享表格「各岗位AI审题进度 → 8月迭代」搭建，让大家**像在共享表格里挂附件一样，在这里上传/下载题目与审核文件，并直观看到每个岗位的进度**。

---

## 一、先看进度（相当于共享表格的主视图）

👉 **[进度看板.md](./进度看板.md)** —— 一张表看全 37 个岗位的当前进度（待出题 / 待一审 / 审核中…），以及每个岗位在各审核轮次的状态（✅ 已完成 / 🔄 进行中 / ⬜ 未开始）和已上传的文件。

> 进度看板是从各岗位已上传的文件推断的初始状态，**之后每次上传/审核完文件，请顺手更新看板里对应的格子**（在 GitHub 网页上点文件 → 铅笔图标即可编辑）。

---

## 二、这是用来干什么的

原共享表格里，每个岗位是一行，每个审核环节（待审核题表、一审、二审……七审、看板汇总、语言审核）的单元格里挂着对应的 Excel 文件。本仓库把这种协作搬到 GitHub：

- **文件夹 = 表格里的「岗位 + 审核环节」**
- **文件 = 原来挂在单元格里的 Excel**
- 上传文件 = 在单元格里加附件；下载文件 = 从单元格下载附件
- **进度看板 = 表格的整体视图**

全程在 GitHub 网页点鼠标即可，不需要会 Git（见第五节）。

目录层级对应表格的前 5 列：`招聘场景 / 行业 / 子行业 / 岗位类型 / 岗位名称`

    ├── 半导体与集成电路/...

## 四、文件命名建议

沿用共享表格的命名习惯：

```
<子行业>-<岗位>题目_待N轮审核.xlsx
2轮_<子行业>_<岗位>_可保留的题目.xlsx / _需要重新出题.xlsx / _重新出答案.xlsx
```

示例：`半导体与集成电路制造业-设备工程师题目_待3轮审核.xlsx`

---

## 五、怎么上传 / 下载（网页操作，零基础）

**上传**：进入目标阶段文件夹（如 `校园招聘/制造业/新能源制造/生产物流类/质量工程师/00_待审核题表/`）→ 右上角 **Add file → Upload files** → 拖入 Excel → 填一句说明 → **Commit changes**。

**下载**：点开文件 → **Download raw file**；或仓库首页 **Code → Download ZIP** 下载全部。

**更新进度看板**：打开 `进度看板.md` → 右上角铅笔图标 → 改对应格子的 ✅/🔄/⬜ → Commit。

**新建岗位/行业**：上传文件时在文件名前加上路径即可自动建文件夹，例如 `互联网/产品类/产品经理/00_待审核题表/题目.xlsx`。

## 六、协作约定（建议）

- 文件名清楚标注「子行业-岗位-轮次」。
- 同一份文件更新时**同名覆盖上传**，GitHub 自动保留历史版本（文件页 **History** 可查看/回退）。
- 每次上传在 commit 说明里写清楚做了什么。
- 上传/审核完记得更新 **进度看板** 对应格子。
- 邀请协作者：仓库 **Settings → Collaborators**。

> 本仓库结构来源：企业微信共享表格「各岗位AI审题进度」的 **8月迭代** sheet（共 41 列、37 个岗位）。如需扩展到 6月/4月迭代或更多行业岗位，按相同层级新建文件夹即可。
