## 一、先看进度（相当于共享表格的主视图）

👉 **[进度看板.md](./进度看板.md)** —— 一张表看全 37 个岗位的进度，按 **第1轮 / 第2轮 / 第3轮…** 逐轮打勾，每轮检查应上传的 5 类文件（待审题表 / 保留题目 / 需重出题 / 需重出答案 / 审核看板）是否已传，已传打 ✅。

> 🤖 **看板会自动更新，无需手动维护**：仓库配置了 GitHub Actions（[工作流](./.github/workflows/update-dashboard.yml)），每当有人往某个岗位的类型文件夹里上传文件，机器人就会自动重算并刷新 `进度看板.md`。
>
> 进度判定规则：脚本按文件名里的轮次前缀（如 `2轮_…`、`待3轮审核`）把文件归到对应轮次，再逐轮检查 5 类是否齐全。

## 二、这是用来干什么的

原共享表格里，每个岗位是一行，每个审核环节（待审核题表、一审、二审……七审、看板汇总、语言审核）的单元格里挂着对应的 Excel 文件。本仓库把这种协作搬到 GitHub：

- **文件夹 = 表格里的「岗位 + 审核环节」**
- **文件 = 原来挂在单元格里的 Excel**
- 上传文件 = 在单元格里加附件；下载文件 = 从单元格下载附件
- **进度看板 = 表格的整体视图**

全程在 GitHub 网页点鼠标即可，不需要会 Git（见第五节）。

## 三、目录结构（已覆盖表格全部列）

目录层级对应表格的前 5 列：`招聘场景 / 行业 / 子行业 / 岗位类型 / 岗位名称`

```
校园招聘/
└── 制造业/
    ├── 汽车及零部件/{生产物流类|技术研发类|软件算法类}/<岗位>/
    ├── 新能源制造/...
    ├── 半导体与集成电路/...
    ├── 消费电子／通信／智能设备/...
    └── 制造业通用/...
通用行业/技术研发岗/工程技术通用/
```

每个**岗位文件夹**下，固定分成 5 个类型子文件夹（按文件类型存放，不再按轮次拆）：

| 类型子文件夹 | 放什么 |
| --- | --- |
| `1_待审核题目` | 各轮「待N轮审核」的题目表 |
| `2_保留题目` | 各轮审核后可保留的题目（统一放，不分轮） |
| `3_需重新出题` | 各轮审核后需要重新出题的清单 |
| `4_需重新出答案` | 各轮审核后需要重新出答案的清单 |
| `5_审核结果看板` | 各轮审核后生成的看板 |

> 不同轮次的文件靠文件名前缀（如 `2轮_…`、`待3轮审核`）区分，进度看板会据此逐轮打勾。
>
> 排期与负责人信息（知识/语言审核时间、分工）集中放在 **进度看板** 里展示。`.gitkeep` 是占位文件，可忽略。

## 四、文件命名建议

沿用共享表格的命名习惯：

```
<子行业>-<岗位>题目_待N轮审核.xlsx
2轮_<子行业>_<岗位>_可保留的题目.xlsx / _需要重新出题.xlsx / _重新出答案.xlsx
```

示例：`半导体与集成电路制造业-设备工程师题目_待3轮审核.xlsx`

## 五、怎么上传 / 下载（网页操作，零基础）

**上传**：进入目标类型文件夹（如 `校园招聘/制造业/新能源制造/生产物流类/质量工程师/1_待审核题目/`）→ 右上角 **Add file → Upload files** → 拖入 Excel → 填一句说明 → **Commit changes**。

**下载**：点开文件 → **Download raw file**；或仓库首页 **Code → Download ZIP** 下载全部。

**进度看板**：不用手动改。上传文件后等一会（约 1 分钟），机器人会自动刷新 `进度看板.md`。

**新建岗位/行业**：上传文件时在文件名前加上路径即可自动建文件夹，例如 `互联网/产品类/产品经理/1_待审核题目/题目.xlsx`；新岗位也会自动出现在看板里。

## 六、协作约定（建议）

- 文件名清楚标注「子行业-岗位-轮次」。
- 同一份文件更新时**同名覆盖上传**，GitHub 自动保留历史版本（文件页 **History** 可查看/回退）。
- 每次上传在 commit 说明里写清楚做了什么。
- **进度看板会自动刷新**，不用手动改；如需立即重算，可在仓库 **Actions** 页手动运行「自动更新进度看板」。
- 邀请协作者：仓库 **Settings → Collaborators**。
